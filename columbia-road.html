Timeline

Overview

The goal of this project was to redesign an existing website page to make it more usable, accessible and responsive across various screen sizes (computer, tablet and mobile). I engaged in the design process, from sketching to high fidelity wireframes, and then used HTML and CSS to code the interface!

I chose to redesign the home page for the Columbia Road Website, which is shown below.

Problem

Why this website?

Columbia Road is often bustling on Sundays with a flower market and local shops. However, the existing Columbia Road website lacks visual hierarchy and is not intuitive to use, which is unfortunate given the vibrant energy in this part of London.

Usability Issues

  • Font weights and sizing inconsistent -> difficult to find information quickly

  • Unclear what text and images are clickable

  • Navigation bar is not sticky

  • When resizing interface, one of shops disappears and leaves empty space

Accessibility Issues

  • Some links hidden behind graphic elements (may be confusing to screenreaders)

  • When resizing interface, one of shops dissapears yet link remains

Style Guide

My style guide allowed me to visualize the colors, fonts, icons and effects of the website. I used earth tones, drawing inspiration from the green of the original website, and a more playful font for the headings to capture the energy of the Columbia Road community.

High Fidelity Wireframes

Using the style guide, I could bring the low-fidelity wireframe to life as a high-fidelity mock up. During this process, I made revisions to my sketches, low-fidelity wireframes and style guide when I wanted to add or remove an element now that I could see the design with styling.

Website

Finally, I coded the website using CSS and HTML, and WebAIM WAVE (an accessibility checker) detected zero errors it. You can visit the website here!

Reflections

This project taught me a lot about the design process and how each step builds on the previous one. I had to not only think critically about how to improve a website but also justify my design decisions in terms of accessibility and usability. In addition, I became more comfortable using flex boxes and coding in relative values rather than fixed values. I am excited to continue using these practices when creating websites in the future!
